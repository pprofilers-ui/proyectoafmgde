Conclusion principal ya confirmada
----------------------------------
Se ha validado con exito que AFMGDE puede consumir correctamente el flujo corporativo AGQ de validacion de credenciales, siempre que se respeten los dos pasos del servicio y la autenticacion tecnica de la API.


Flujo real confirmado
---------------------
El flujo correcto NO es un unico endpoint.

Se compone de dos llamadas:

1. Primer servicio
- GET /api/test/validarCredenciales/{idAplicacion}/{Usuario}/{Password}
- requiere credencial tecnica Basic de la API
- devuelve:
  {
    "Message": "...."
  }

2. Segundo servicio
- POST /api/usuarios/validarCredenciales/{idAplicacion}
- requiere la misma credencial tecnica Basic de la API
- body:
  {
    "Message": "...."
  }
- devuelve:
  {
    "Response": true
  }


Autenticacion tecnica de la API
-------------------------------
Para consumir los servicios NO basta con usar solo el usuario/password AGQ.

Tambien hace falta la credencial tecnica de la API, observada en Swagger.

Valor probado con exito:
- api_basic = c2lsX2FwaTpoalJBU3NAZXc1T3JPNw==

Esto significa que existen dos capas:

1. credencial tecnica para acceder a la API
2. credenciales del empleado AGQ que se quieren validar

Conclusion tecnica importante
-----------------------------
No hace falta descifrar manualmente Message dentro de AFMGDE.

La logica correcta es:

1. pedir Message al primer servicio
2. reenviar Message al segundo servicio
3. usar Response=true/false como resultado final

Esto simplifica mucho la integracion futura.


Servicio interno ya preparado
-----------------------------
La clase:
- users/agq_auth_service.py

ya encapsula este flujo y devuelve un resultado estructurado.

Su responsabilidad actual es:

- generar Message
- validar Message
- devolver si las credenciales son validas
- sin tocar login Django real


Lo que NO se ha hecho todavia
-----------------------------

- creacion automatica de sesion Django desde Response=true
- mapeo automatico de empleado AGQ a usuario local Django
- fallback de login local/admin
- gestion final de errores visuales para el usuario
